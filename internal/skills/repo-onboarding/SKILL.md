# repo-onboarding

## Workflow

1. 先给出目录级地图（顶层、核心模块、测试目录）。
2. 确认启动与构建命令，以及依赖管理方式。
3. 给出关键执行路径（入口 -> 主要模块 -> 输出/副作用）。
4. 最后列出后续深入建议（读哪些文件、跑哪些命令）。

## Output Contract

- Repo Map
- Build/Test Entry
- Core Runtime Flow
- Next Read/Run Checklist
